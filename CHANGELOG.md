# 更新日志

本文件记录每个版本的用户可见变更。发布前必须为新版本添加 `## vX.Y.Z` 小节——
release 工作流会提取对应小节作为 GitHub Release 的说明（缺失则构建失败），
应用内的更新弹窗和「更新完成」日志展示的也是这段内容。

## v1.0.5 - 2026-06-14

### 修复
- 修复鼠标悬停待办 / 日程事件时提示框看不清的问题：原生提示框会继承该行的文字颜色，导致磨砂皮肤下「黑底黑字」、液态玻璃皮肤下顶部出现一层白色遮罩。现改用浅底深字的气泡提示，两种皮肤下都清晰可读
- 加固应用内自动更新：更新助手改为独立的 Python 进程（不再依赖 PowerShell）；当应用未能在超时内退出时，不再覆盖安装到运行中的程序、也不会重复启动两个实例；更新若未成功，下次启动会明确提示「上次更新失败」
- 系统托盘菜单改为从光标位置向左上方弹出，并限制在屏幕可视区域内，不再被屏幕底边裁切

### 改进
- 设置窗口进一步放大（1120×860），分类导航、设置项标题、控件与说明文字的字号整体调大，更易读
- 「添加事项」与「设置截止时间」弹窗放大、输入框字号调大，输入更舒适

### 其他
- 代码重构与质量保障：拆分 app.py 为多个 UI 模块，更新器逻辑改为无 Qt 依赖且可单元测试；新增测试套件并在发布前作为 CI 门禁运行，降低回归风险（对用户无可见变化）

## v1.0.4 - 2026-06-12

### 新功能
- 多日历订阅：可添加多个 ICS / webcal 链接，每个日历前有勾选框，勾选的日历的日程才在备忘录窗口中展示
- 订阅链接自动存档：删除的链接保留在「恢复已删除」菜单中，可一键找回；日历名称自动从订阅源读取
- 设置窗口支持按住空白处拖动

### 修复
- 「恢复默认」不再清除日历订阅和行为设置：现仅恢复「外观」分类的默认值（按钮已改名为「恢复默认外观」）
- 多个订阅中某一个同步失败时，不再影响其他日历的日程展示

## v1.0.3 - 2026-06-12

### 修复
- 修复应用内自动更新的竞态问题：旧逻辑固定等 2 秒就启动安装器，应用退出稍慢时会被安装器强行关闭并弹出报错。现在更新助手会等待应用进程真正退出后再静默安装，抑制安装器报错弹窗，且无论安装成功与否都会自动重启应用；安装日志写入 `%TEMP%\LiquidMemoWidget-update.log` 便于排查

### 改进
- 设置窗口改为左右两栏布局：左侧分类导航（外观 / 行为 / 日历订阅 / 关于），右侧显示对应分类的设置内容
- 设置项的说明文字收纳到标题旁的 ⓘ 图标中，鼠标悬停即弹出解释气泡，界面更紧凑
- 设置界面整体放大：窗口 960×720，行标题、导航、下拉框、输入框、开关、按钮和说明气泡的字号全面调大，更易读
- 系统托盘菜单字号由 15px 调大到 17px

## v1.0.2 - 2026-06-11

### 修复
- 修复 v1.0.1 安装版启动即崩溃（`ModuleNotFoundError: No module named 'xml.etree'`）：打包配置遗漏了标准库 `xml.etree`，现已显式打入安装包

## v1.0.1 - 2026-06-11

### 修复
- 修复「检查更新」在共享出口 IP 网络下报 `HTTP Error 403: rate limit exceeded` 的问题：GitHub API 被限流时自动降级到不限流的 releases.atom 订阅源，版本检查、更新说明获取和安装包下载均不再依赖 API 配额

## v1.0.0 - 2026-06-11

### 新功能
- 应用内自动更新：启动后静默检查 GitHub Releases，发现新版本时弹出更新卡片，展示更新说明，一键下载（带进度）、静默安装并自动重启
- 更新完成后首次启动自动展示一次该版本的更新日志
- 设置窗口新增「关于」分区：GitHub 仓库链接和「检查更新」按钮
- 全新应用图标，安装包、可执行文件、系统托盘和窗口统一使用

### 改进
- 发布流程自动把 tag 版本号写入应用（应用内可见准确版本）
